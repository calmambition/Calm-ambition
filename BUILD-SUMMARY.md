# Burnout Check-In Build Summary

## What's Built

You now have a production-ready AI-powered burnout assessment that replaces your basic quiz with something that drives real lead generation value.

### 1. The Check-In Page (`/check-in/index.html`)

Vanilla JavaScript port of the React prototype. No build step needed—it runs directly on GitHub Pages.

**Flow:**
- Intro (what the quiz measures, no data collection disclaimer)
- 9 scale questions (Exhaustion, Detachment, Efficacy dimensions)
- 2 free-text questions (context about their situation)
- Email gate (collects name + email before showing results)
- Personalized AI readout (generated by Claude, tailored to their answers)
- Three concrete action steps specific to their situation
- CTA to book a discovery call

**Key features:**
- Smooth animations (rise, breathe, progress bar)
- Mobile-first responsive design
- Full keyboard navigation (accessible)
- No data storage (answers aren't kept after readout)
- Google Analytics event tracking built in

### 2. Cloudflare Worker (`worker.js`)

Handles all API calls server-side so your Anthropic API key never touches the browser.

**What it does:**
- Validates incoming answers (must be 9 integers, 0–4)
- Validates free text (max 1,000 chars per field)
- Builds the LLM prompt server-side
- Calls Anthropic API with your key
- Rate limits to 10 requests/hour per IP
- Returns clean JSON readout or error
- CORS locked to your site origin only

**Security:**
- API key stored in Cloudflare Secrets (never in code)
- No logging of personal answers
- Rate limited to prevent abuse
- CORS prevents anyone else from using your worker endpoint

### 3. Wrangler Config (`wrangler.toml`)

Ready to deploy. You'll set the KV namespace ID and API key during setup.

### 4. Email Integration

Readouts are sent via Formspree (your existing endpoint). The check-in page submits the email address + readout JSON after someone enters their email on the gate.

Currently sends raw JSON. Can be beautified later if needed.

## Architecture Diagram

```
Browser (GitHub Pages)
  check-in/index.html
    ├─ Collects 9 answers + 2 free-text responses
    ├─ POSTs to Cloudflare Worker
    └─ Displays personalized readout

Cloudflare Worker
  ├─ Validates input (9 ints 0–4, text <1000 chars)
  ├─ Builds prompt with intervention library
  ├─ POSTs to Anthropic API with secret key
  ├─ Rate limits per IP (KV namespace)
  └─ Returns JSON readout

Anthropic API
  └─ claude-sonnet-4-20250514
      Input: 9 answers + dimension scores + free text
      Output: headline + 3 dimensions + pattern + 3 steps + invitation

Email (Formspree)
  └─ Sends readout to user's email with CTA
```

## Support Notes

If anything goes wrong:

- **Worker deployment fails?** Make sure you're authenticated with Wrangler (`wrangler login`)
- **API key not working?** Verify it's from https://console.anthropic.com/account/keys (not your API key for another service)
- **Quiz shows API error?** Check the worker URL is correct in the check-in page
- **Email not arriving?** Check Formspree dashboard for the submission. Look in spam folder.
- **Stale content on mobile?** Service worker caching. Hard refresh or clear app cache.

All of these are fixable. None are show-stoppers.
